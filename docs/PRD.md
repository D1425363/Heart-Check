# PRD: Heart Check 校園互助與人氣回饋平台

## 1. 專案概述
### 背景與動機
在校園生活中，學生經常會遇到需要幫助的情況（如：尋找失物、課業疑問、生活協助等），而許多熱心的同學也願意伸出援手。然而，缺乏一個良好的正向回饋機制，導致許多互助行為僅止於單次事件，難以形成持續的校園互助文化。此外，校園內部的實用資訊（如：交通、宿舍、餐廳）常散落在不同平台，缺乏整合。本專案希望能透過「愛心值」與「人氣回饋」機制，鼓勵學生互相幫助，並整合校園生活資訊，建立一個更便利且充滿人情味的校園環境。

### 目標用戶
- **主要用戶**：在校學生（包含有求助需求者及願意提供幫助者）。
- **次要用戶**：學生會、宿舍管理員等（負責發布公告與生活資訊）。

### 核心價值主張
- **正向激勵**：透過「愛心值」與「人氣累積」機制，讓幫助他人的行為得到實質認可與記錄。
- **便捷互動**：透過 QR Code 快速進行互動與發送愛心值，降低互助與回饋的門檻。
- **資訊整合**：提供一站式的校園生活資訊平台，涵蓋失物招領、交通、宿舍與餐廳等資訊，解決日常痛點。

## 2. 功能需求

### 1. 會員與個人檔案系統
- **作為** 學生，**我希望** 能夠註冊並建立個人檔案，**以便** 查看我的愛心值餘額與累積人氣。
- **作為** 學生，**我希望** 能夠產生專屬的 QR Code，**以便** 他人可以快速掃描並發送愛心值給我。

### 2. 愛心值與 QR Code 回饋機制
- **作為** 受助者，**我希望** 能掃描幫助者的 QR Code 並輸入愛心值數量與感謝語，**以便** 即時回饋並表達感謝。
- **作為** 系統使用者，**我希望** 能查看愛心值/人氣排行榜，**以便** 了解校園內的熱心同學並獲得見賢思齊的激勵。

### 3. 失物招領功能
- **作為** 遺失物品的學生，**我希望** 能發布尋物啟事（含照片、描述、遺失地點），**以便** 請同學協尋。
- **作為** 拾獲物品的學生，**我希望** 能發布拾獲公告，**以便** 物品主人能盡快聯繫我並取回物品。

### 4. 宿舍與校園公告看板
- **作為** 學生，**我希望** 能查看宿舍與學校的最新公告，**以便** 掌握重要活動、維修資訊或規定變更。
- **作為** 管理員/幹部，**我希望** 能新增、編輯或刪除公告，**以便** 有效傳遞資訊給全體同學。

### 5. 交通與餐廳資訊整合
- **作為** 學生，**我希望** 能查詢校園周邊的交通資訊（如：校車時刻），**以便** 規劃通勤時間。
- **作為** 學生，**我希望** 能查看學餐或周邊餐廳的資訊與評價，**以便** 決定用餐地點。

## 3. 非功能需求

### 技術限制
- **前端技術**：HTML, CSS, JavaScript 搭配 **Jinja2** 模板引擎進行頁面渲染。
- **後端技術**：使用 Python 搭配 **Flask** 框架開發。
- **資料庫**：使用 **SQLite** 儲存使用者資料、愛心值紀錄、公告與失物招領等資訊。

### 效能與安全考量
- **效能考量**：
  - 資料庫查詢應建立適當的 Index，以提升列表（如公告、失物招領）讀取速度。
  - 行動裝置為主：因依賴 QR Code 掃描，網頁需具備良好的行動裝置響應式設計 (RWD)，確保手機載入與操作順暢。
- **安全考量**：
  - 密碼需進行 Hash 處理（如使用 bcrypt）後儲存，絕不以明碼存放。
  - 防範常見的 Web 攻擊，如 XSS（透過 Jinja2 預設的跳脫機制）與 SQL Injection（透過 ORM 或 Parameterized Query）。
  - API 與敏感操作（如發送愛心值）需進行權限驗證，確保僅有登入使用者可進行操作。

## 4. MVP 範圍

### Must Have (必須具備 - 核心價值)
- 使用者註冊、登入與個人檔案頁面。
- 個人專屬 QR Code 產生與顯示功能。
- 掃描 QR Code 轉移「愛心值」的功能（包含感謝留言紀錄）。
- 基礎的失物招領佈告欄（發文、瀏覽）。

### Should Have (應該具備 - 提升體驗)
- 愛心值排行榜（總榜/月榜）。
- 宿舍與校園公告看板。
- 行動裝置完美適配 (Mobile First 設計)。

### Nice to Have (可以具備 - 未來擴充)
- 校園交通資訊（靜態時刻表或串接動態 API）。
- 校園餐廳資訊與簡易評價留言系統。
- 站內信或即時通知系統，方便失物招領聯繫。
